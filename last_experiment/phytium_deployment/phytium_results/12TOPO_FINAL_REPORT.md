# PQ-NTOR 12拓扑性能测试报告 - 飞腾派ARM64

**测试时间**: 2025-11-29
**测试平台**: 飞腾派 (ARM Cortex-A72)
**测试方法**: 单机模拟，每个拓扑100次握手
**PQ算法**: Kyber-512 KEM

---

## 📊 核心发现

### 飞腾派ARM64性能总览

| 指标 | 数值 |
|------|------|
| **平均握手时间** | **180.75 µs** (0.18 ms) |
| **性能范围** | 179.28 - 182.41 µs |
| **最大波动** | 3.13 µs (1.7%) |
| **标准差** | < 1 µs |
| **测试拓扑** | 12个SAGIN场景 |

### 关键观察

✅ **性能极度稳定**
- 12个拓扑的握手时间差异仅1.7%
- 网络参数(速率、延迟、丢包)对单机测试影响微小
- 说明性能主要取决于CPU计算能力

✅ **性能完全实用**
- 180 µs = 0.18 ms
- 真实SAGIN网络延迟(5-10 ms)远大于握手时间
- 握手不会成为性能瓶颈

### 3. 丢包率影响 (0.1 - 2.0%)

| 丢包类型 | 拓扑 | 丢包率 | 握手时间 | 差异 |

**结论**: 丢包率对握手时间**无影响** (单机环境特性)

### 总结

在单机测试环境下:
- **网络参数影响**: 微小 (< 2%)
- **主导因素**: CPU计算性能
- **性能瓶颈**: Kyber-512密码运算

### 1. 性能指标

| 指标 | 数值 | 评价 |
|------|------|------|
| 平均握手时间 | 180 µs | ⭐⭐⭐⭐ 优秀 |

| 计算开销 | 0.18 ms | ⭐⭐⭐⭐ 可接受 |

### 2. 与WSL对比 (估算)

根据之前的单次benchmark测试:

| 平台 | 架构 | 握手时间 | 性能比 |
|------|------|---------|--------|
| WSL2 | x86_64 | ~31 µs | 1.0x (基准) |
| 飞腾派 | ARM64 | ~180 µs | 5.8x 慢 |

**原因分析**:
- CPU架构差异 (x86 vs ARM)
- 主频差异 (x86高主频 vs ARM低功耗)
- 密码运算优化 (x86 AVX2 vs ARM NEON)

### 3. 应用场景适用性

| 场景 | 适用性 | 理由 |
|------|--------|------|
| **SAGIN卫星节点** | ⭐⭐⭐⭐⭐ | 低功耗,180µs完全可接受 |
| **UAV无人机** | ⭐⭐⭐⭐⭐ | 低功耗,体积小 |
| **地面移动节点** | ⭐⭐⭐⭐ | 性能足够 |
| **高性能服务器** | ⭐⭐⭐ | x86更优 |
| **边缘计算** | ⭐⭐⭐⭐⭐ | 成本低,功耗低 |
